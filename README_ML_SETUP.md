# 3D Printer Health Monitoring System with ML

This is a machine learning-based monitoring system for 3D printer health, detecting normal operation vs. creep behavior using real-time sensor data.

## 📊 Project Structure

```
├── data/
│   ├── healthy_data.csv      # Training data for healthy operation
│   ├── creep_data.csv         # Training data for creep behavior
├── src/
│   ├── test.cpp               # Original sensor reading code
│   └── test_with_ml.cpp       # Updated code with ML predictions
├── ml_train.py               # Python script to train ML model
└── requirements.txt          # Python dependencies
```

- Extract model parameters from the Python training output
- Implement in Arduino using the normalized parameters shown

| DHT11 (Nozzle Temp) | D8 | dht1 |
| DHT11 (Bed Temp) | D9 | dht2 |
| Vibration Sensor | D7 | vibCount |
| ACS712 Current Sensor | A0 | motorCurrent |

## 🎯 Classification Logic

The system responds with three states:

### HEALTHY ✅
- Nozzle Temp < 225°C
- Bed Temp < 70°C
- Motor Current < 1.8A
- Vibrations < 70/min

### CREEP ⚠️
- Nozzle Temp > 235°C **OR**
- Bed Temp > 72°C **OR**
- Motor Current > 2.5A **OR**
- Vibrations > 90/min

### CAUTION 🟡
- Between healthy and creep thresholds

## 🔧 Customization

### To Update Thresholds
Edit these constants in `src/test_with_ml.cpp`:
```cpp
const float HEALTHY_NOZZLE_MAX = 225.0;
const float CREEP_NOZZLE_MIN = 235.0;
// ... etc
```

### To Add More Training Data
1. Collect real sensor data in CSV format
2. Add rows to `data/healthy_data.csv` or `data/creep_data.csv`
3. Re-run `python ml_train.py`
4. Update thresholds based on new results

### To Add New Sensors
1. Add sensor reading code in `loop()` function
2. Include in classification logic
3. Update CSV files with new feature
4. Retrain model with `ml_train.py`

## 🐛 Troubleshooting

### Sensor reads "nan"
- Check DHT wiring (power, GND, data pin)
- Verify pull-up resistor on data line

### Inaccurate current readings
- Calibrate `zeroVoltage` in code
- Ensure ACS712 is properly powered
- Check sensitivity value for your module (mV/A)

### OLED not displaying
- Verify I2C address (0x3C assumed)
- Check SDA/SCL connections
- Install Adafruit libraries from PlatformIO

## 📚 Files Generated by Training

When you run `python ml_train.py`, it creates:
- `model/scaler.pkl` - Feature normalization parameters
- `model/model.pkl` - Trained Random Forest classifier

These can be used programmatically but aren't needed for the Arduino threshold-based approach.

- Vibration Sensor (normally open contact)

## 📝 CSV Data Format

Both `healthy_data.csv` and `creep_data.csv` must have:
```
nozzle_temp,bed_temp,current,vib_per_min,status
210.5,60.2,1.2,45,healthy
```

Where:
- `nozzle_temp`: Temperature in °C (float)
- `bed_temp`: Temperature in °C (float)
- `current`: Motor current in Amperes (float)
- `vib_per_min`: Vibrations per minute (integer)
- `status`: "healthy" or "creep" (string)
